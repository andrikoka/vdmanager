Dokumentációkkal kapcsolatos követelmények

Fejlesztői dokumentáció

A fejlesztői dokumentációt szabványos HTML formátumban vagy egyszerű szöveg formátumban kell megírni és tárolni. A fájl karakterkódolása minden esetben UTF-8 vagy ISO-8859-2 kell, hogy legyen. HTML esetén a szabványos HTML-nek megfelelően, stílusinformációk nélkül kell létrehozni, csak külső, közös CSS-el formázva. A dokumentáció struktúrája a Software követelmény specifkációban lesz részletesen definiálva.

Felhasználói dokumentáció

A felhasználói dokumentációnak a szoftveren belül kell helyet foglalni, elérhetővé tenni. A Súgó-rendszernek minden operációs rendszeren egységesnek, kereshetőnek kell lennie. A szoftveren belül helyet foglaló helyzetérzékeny súgók, eszköztippeknek* minden esetben elérhetőnek kell lenniük.
A felhasználói dokumentációt szabványos HTML formátumban kell tárolni a szoftveren belül, annak frissítéséért és naprakészen tartásáért az arra kijelölt fejlesztő felelős. A dokumentáció olvasására, abban való keresésére a grafikus modulon belül egy külön részt kell létrehozni, elkerülve az operációs rendszerek beépített súgórendszereit.
A súgórendszer és a dokumentáció egy és ugyanazon rendszer lesz, így minden esetben konzisztensen, könnyen átlátható struktúrában kell tartani. Mivel ez a szoftver első verziója, így ezen dokumentáció első példányának megírásakor kiemelt figyelmet kell fordítani a pontos és érthető funkcióleírásokra.
